# T061 自动化脚本配置指南

**目标：** 一键上传 15 张图片到 Supabase + Airtable

---

## 📋 前提条件

- 电脑装有 **Node.js**（v14+）
- 15 张 LoveArt 生成的 JPG 图片
- Supabase 账户 + Airtable 账户

---

## 🔧 环境配置（5 分钟）

### **第 1 步：获取 Supabase 凭证**

1. 打开 Supabase Dashboard
2. 左侧 **Settings** → **API**
3. 复制以下信息：
   - **Project URL**（例如：`https://xxxxx.supabase.co`）
   - **anon public key**（例如：`eyJh...`）

### **第 2 步：获取 Airtable 凭证**

1. 打开 airtable.com，登录
2. 打开你的 Base
3. 右上角 **Account** → **Develop** → **API tokens**
4. 创建新 token：
   - 权限勾选：`data.records:read`, `data.records:write`
   - 复制 token
5. 同时需要 Base ID：
   - 打开任意一个 Base 的 URL：`https://airtable.com/{BASE_ID}/...`
   - 复制 BASE_ID 部分

### **第 3 步：创建 .env 文件**

在脚本所在的文件夹中，创建文件 `.env`：

```env
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_KEY=eyJhbGci...（你的 anon key）
AIRTABLE_TOKEN=pat_xxxxx...（你的 API token）
AIRTABLE_BASE_ID=appxxxxx...（你的 Base ID）
```

**⚠️ 重要：不要把 .env 上传到 GitHub！**

### **图片命名规范**（重要！）

脚本根据文件名来匹配 Airtable 记录，所以名称必须准确：

```
apr{日期}__描述.jpg

例如：
✅ apr21__shanghai_reel.jpg      （正确）
✅ apr25__beijing_xian_feed.jpg  （正确）
❌ apr21_shanghai.jpg            （错误 - 缺少 __）
❌ shanghai_reel.jpg             （错误 - 缺少日期）
```

```bash
# 安装依赖
npm install @supabase/supabase-js node-fetch dotenv
```

## 🐛 常见错误解决

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `Cannot find module '@supabase/supabase-js'` | 缺少依赖 | 运行 `npm install` |
| `SUPABASE_URL is required` | 没有配置 .env | 检查 .env 文件是否存在且格式正确 |
| `Airtable API 错误：401` | token 错误或过期 | 重新生成 token，更新 .env |
| `未找到匹配的 Airtable 记录` | 文件名与日期不匹配 | 检查文件名是否符合 `apr21__描述.jpg` 格式 |
| `Supabase 上传失败` | bucket 名称或权限错误 | 检查 bucket 是否存在、是否为 public |
