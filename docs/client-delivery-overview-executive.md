## 二、页面与内容规模（甲方最关心的「有多少页」）

### 2.1 怎么理解「页面数」

网站上既有**一篇一篇能点开的文章/指南**，也有**一条条独立产品行程**，还有**活动专题页**、**工具页**等。为便于沟通，分三层说明：

1. **固定结构与 SEO 内容**（旅游指南、枢纽页、签证/季节专题等）
  内部站点结构文档口径为 **「62+」个主要内容单元**（可理解为：访客在站内能稳定到达的**大量独立内容页与枢纽页**的组合规模）。
2. **行程产品**
  每条线路有**独立详情页**（含亮点、日程、包含/不包含项等），并支持**单独打印版行程单**。当前数据集中维护的行程产品规模为**数十条级别**（具体条数会随业务上新/调整而变化）。
3. **活动与战役**
  例如「十月 Discovery」专题：**索引页 + 多条活动落地页**，并与对应产品线路在话术与链接上对齐，便于投放与社媒统一收口。

**一句话：** 对客侧是「**整站品牌官网 + 大规模目的地内容 + 可售卖产品库 + 战役专题**」的组合，而不是少数几个静态介绍页。

## 三、现在网站「都能做什么」（用业务语言描述）

访客与顾问日常会用到的能力，概括如下：

- **品牌形象与获客入口**：首页、关于我们、联系方式、条款等基础信任页。  
- **产品与转化**：全部行程浏览、按目的地/系列浏览、每条行程的详情展示；关键按钮在跳转时有**可见的加载反馈**，减少客户以为「点了没反应」的体验问题。  
- **行程材料输出**：单条行程支持**浏览器打印版**；支持访客留下邮箱**申请发送行程**（邮件通道需在托管平台配置密钥后方可稳定投递）。  
- **内容与 SEO**：面向新西兰读者的**全国性枢纽页**、各城市/区域**枢纽页**、**最佳季节**与**中国入境/签证信息（面向新西兰护照读者）等；另有大量目的地与景点深度指南**（便于自然搜索与长周期种草）。  
- **市场与战役**：十月等**专题活动页**，文案与日期可随战役节奏调整；与主站产品路径一致，便于广告与社媒使用统一链接。  
- **顾问与渠道支持**：代理页、定制行程入口等（以当前线上导航为准）。  
- **辅助工具与扩展阅读**：旅行工具、博客、美食指南等模块（持续可扩展）。  
- **内部协作（不占对客主线）**：营销战役看板、社媒发帖排期表等，用于**团队内部对齐与验收**，不替代对外官网。

## 四、相对「最初一版官网」——我们**额外**多做了哪些（价值向）

下列多项在典型「先做官网骨架」的合同里**常常不会第一版就包齐**，但在本项目迭代中已经落地或具备可用形态，建议作为**增量成果**理解：

1. **大规模 SEO 内容矩阵**
  不仅是首页与行程列表，而是系统性建设**枢纽页 + 目的地/景点长文指南**，形成可长期获客的「内容资产」。
2. **战役级落地与口径统一**
  专题活动页、与 Discovery 产品线的对应关系、旧链接到新主推产品的**平滑迁移**（减少广告与旧物料失效）。
3. **图片与展示稳定性**
  将大量图片纳入可托管、可缓存的云端存储方案，解决「线上图裂、慢、路径乱」类问题，属于**上线质量类硬投入**。
4. **体验细节**
  导航与关键按钮的加载反馈、打印与邮件等「销售顾问日常会用到」的流程型能力。
5. **内部营销工具**
  战役看板、社媒发帖计划表等，让市场同事有**可执行的排期与链接清单**，减少微信群碎片化信息。
6. **持续迭代节奏**
  在数周内以高频提交合并的方式持续修复与增强，而不是一次性交付后长期停滞。

## 五、给您做决策用的三句结论

1. **规模：** 对客是「品牌官网 + 大量内容页 + 数十级行程产品 + 活动专题 + 工具/博客扩展」的一体化站点，**页面与 URL 规模达到「六十以上主要内容单元」这一档**，并叠加每条行程的详情与打印等变体。
2. **时间：** 自 **2026-03-07** 首版入库至 **2026-04-14** 左右，约 **5–6 周**高频迭代，**约 150 次**提交；累计专业工时粗估 **一百至二百小时量级**（见第一节脚注）。
3. **增量：** 在「能卖的官网」之上，额外完成了**内容矩阵、战役落地、图片与体验治理、内部营销协作面**等一大块工作——这也是后续若进入**稳定内容产出与广告投放执行**，需要单独谈服务包或加时的原因。
