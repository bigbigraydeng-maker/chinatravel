## 如何使用

1. 正文须以 `## 标题` 分节；**第一个 `##` 之前不要写正文**（不会被解析进页面）。
2. 在下方 **会议纪要** 粘贴会议原文，或让 AI 先整理成要点再粘贴。
3. 在 **待办**、**开发事项** 使用 Markdown 任务列表：`- [ ]` 未完成，`- [x]` 已完成。
4. 保存后执行 `git commit` → `git push`，Render 部署完成后打开页面即可投屏或共享链接。

（可选）把 AI 输出的整理提示词放在单独一节，便于下次复制。
