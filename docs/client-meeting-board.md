## 如何使用

1. 正文须以 `## 标题` 分节；**第一个 `##` 之前不要写正文**（不会被解析进页面）。
2. **YAML 头**：`lastUpdated`、`subtitle`、**`nextMeeting`**（下次会议时间/地点说明，会显示在看板顶部）。
3. 会后：在 **会议纪要** 写要点；把结论拆成 **待办**（商务/双方）与 **开发事项**（技术交付）。
4. **周间**：在 **定期跟进** 用 Markdown 记电话/邮件结果（可带日期小标题）。
5. **下次会前**：在 **会前 Follow-up** 列 `- [ ]` 核对项（从待办/开发里挑要在会上过的）；会上勾 `- [x]`，会后把结论写回纪要或定期跟进。
6. `git commit` → `git push`，部署后页面刷新。
